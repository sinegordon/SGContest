# DjangoContestClient

Минимальное Django-приложение, повторяющее основной сценарий `PythonContestClient`:

- загрузка пользователя
- выбор курса
- получение набора задач из `ContestServer`
- выбор задачи из реально назначенного набора
- случайный выбор варианта задачи при первичной выдаче студенту
- загрузка файла с решением
- асинхронная отправка решения без перезагрузки страницы
- выбор языка проверки в момент отправки решения
- сброс сессии

Дополнительно есть staff-only раздел `/ops/` для администрирования `ContestServer`:

- просмотр курсов
- просмотр пользователей
- просмотр задач курса
- создание курса
- создание пользователя
- очистка курса
- настройка количества задач уровней `1`, `2`, `3` отдельно для каждого курса
- загрузка информации о курсе для подсчёта реальных лимитов по уровням
- сброс назначенного набора задач у студента по конкретному курсу
- пакетный импорт задач из JSON
- получение дампа сообщений как JSON-файла

Особенности текущего интерфейса:

- адрес сервера скрыт на клиентской странице и берётся из сессии или `CONTEST_SERVER_URL`
- язык не выбирается при загрузке пользователя и задаётся только при отправке решения
- вариант задачи не показывается студенту и выбирается случайно один раз при первичной выдаче
- результаты ops-операций открываются на отдельной странице `/ops/results/`

## Запуск

```bash
cd DjangoContestClient
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python manage.py migrate
python manage.py createsuperuser
python manage.py runserver
```

После запуска открой:

- `http://127.0.0.1:8000/`
- `http://127.0.0.1:8000/admin/`
- `http://127.0.0.1:8000/ops/`

После входа staff-пользователем будет доступен административный раздел:

- `http://127.0.0.1:8000/ops/`

## Запуск через Docker Compose

Из корня репозитория:

```bash
docker compose up --build django-client
```

Сервис поднимет Django, выполнит `migrate`, создаст superuser при первом старте и запустит приложение на:

- `http://127.0.0.1:8000/`
- `http://127.0.0.1:8000/admin/`
- `http://127.0.0.1:8000/ops/`

Для Docker Compose заранее настроен superuser:

- username: `sinegordon`

Переменные superuser читаются из корневого `.env`.

Также через `docker-compose` заранее задаётся:

- `CONTEST_SERVER_URL=http://contest:8000`

То есть Django-клиент внутри compose-сети обращается к `ContestServer` по внутреннему адресу контейнера.
